# 博客写作指南

本文档将指导你如何按照规范编写和发布新的博客文章。

## 1. 元数据格式

每篇博客文章的开头必须包含一个 HTML 注释块，用于定义文章的元数据。这个注释块必须位于文件的前几行。

### 1.1 必填字段

- **title**: 文章的标题。
- **categories**: 文章分类。注意：只能填写**一个**分类。格式为 JSON 字符串数组，例如 `["教程"]`。
- **tags**: 文章标签。可以有多个。格式为 JSON 字符串数组，例如 `["指南", "Markdown"]`。
- **cover_image**: 封面图片路径。目前尚未实现，请留空。
- **summary**: 文章摘要，将显示在文章列表页。

### 1.2 示例


## 2. 内容撰写

博客内容使用标准的 **Markdown** 语法。

### 2.1 标题与书签

系统支持使用 Markdown 的标题语法作为书签（目录索引）：

- `#` 一级标题
- `##` 二级标题
- `###` 三级标题

这些标题将被自动解析并生成文章的目录结构，方便读者跳转。

### 2.2 其他语法

你可以自由使用 Markdown 支持的其他语法，如：

- **粗体** 和 *斜体*
- [链接](https://example.com)
- 代码块
- 列表

## 3. 发布流程

文章编写完成后，需要更新索引文件以使其在博客中可见。

### 3.1 构建索引

请在项目根目录下执行以下 Python 脚本：

```bash
python scripts/build_posts.py
```

该脚本会扫描 `posts` 目录下的所有 `.md` 文件，解析元数据，并生成 `data/posts.json` 索引文件。

### 3.2 验证

构建完成后，刷新博客页面即可看到新的文章。
